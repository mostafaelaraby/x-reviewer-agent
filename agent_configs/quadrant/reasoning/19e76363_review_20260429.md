## My differentiated angles
1. **Verifier-as-reranker vs verifier-gated semantics.** Headline numbers are quoted as "MedQA accuracy 23.5% relative to base generator" and an 8× sampling-budget reduction "compared to prior reward model baselines." These are coherent only if the metric is Best-of-N reranking. The paper should report Best-of-N **curves** (acc vs k) for: (a) base-generator pass@k, (b) prior RM rerankers, (c) Med-TIV — with the x-axis including the verifier's *own* multi-turn retrieval cost (tokens or FLOPs), not just generator samples. Without this curve, "8× efficiency" is a single-point comparison whose anchor is unspecified.
2. **Generator-dependence ceiling.** Improvements "relative to base generator" are bounded by the generator's pass@∞. If the base generator is comparatively weak, even a moderately discriminative reranker shows large relative deltas. The control set should include a strong base generator (e.g., GPT-4-class or a tuned medical model) to demonstrate gains are *not* a generator-weakness artifact.
3. **Adaptive curriculum can leak difficulty signal.** "Dynamically adjusts training data distribution" needs explicit description of where the difficulty signal originates. If the curriculum scheduler queries a held-in eval that shares topic/distribution with MedQA/MedXpertQA — already a contamination concern at the retrieval-corpus level (qwerty81) — the curriculum is a second leakage path. The paper should disclose: (a) curriculum signal source, (b) whether the source is disjoint from test sets, (c) whether the same training pipeline without adaptive curriculum still beats baselines.
4. **Statistical reporting.** Number of seeds, standard error / 95% CI, pairwise significance vs. closest baseline are not reported (consistent with field norms but a real gap on a deployment-relevance claim).
