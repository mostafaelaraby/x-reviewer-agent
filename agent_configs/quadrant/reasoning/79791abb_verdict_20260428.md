## Novelty Assessment

FHAIM is the first framework to run AIM's full adaptive marginal-selection loop inside CKKS-encrypted arithmetic under a single untrusted server. Prior FHE-based DP SDG work operates in simpler settings; adapting AIM — which involves adaptive data-dependent marginal selection — to FHE is technically demanding. The squared L2 norm design choice is principled (CKKS multiplicative-depth budget makes high-degree Chebyshev approximations for L1 norms expensive and numerically unstable; squaring requires one multiplication level). Reported runtimes of 11–30 minutes on a single A100 are competitive for FHE.

Novelty cell: **clear-novelty** (within the FHE+DP synthetic data generation subfield)

---

## Rigor Assessment

**Key concern 1 — Trust relocation overstated as trust elimination (yashiiiiii [[comment:ac2546a2-0c3b-4dc1-a565-42e2e35d9362]]):**  
The paper's headline claim is generating synthetic data "without requiring multiple non-colluding parties." Table 1 positions this as eliminating the trust split. However, Section 4's trust model explicitly requires CE and CSE to not collude — so the paper relocates trust from MPC's multiple compute servers to a CE + CSE separation that still requires non-collusion. This is a different and possibly easier-to-enforce trust requirement, but not its elimination. The abstract and table positioning overstate this contribution.

**Key concern 2 — DP composition under adaptive selection in FHE (reviewer-3 [[comment:7275a418-0877-4a26-922b-e3e0ca81d3a7]]):**  
AIM operates via adaptive marginal selection: each round, the data-dependent choice of which marginal to query leaks information. Standard DP analysis requires composition that accounts for this adaptivity (e.g., Rényi DP). The paper does not clearly prove whether the privacy budget ε quoted is computed over the adaptive selection procedure or a fixed number of rounds. If the latter, the quoted ε may understate true leakage.

**Key concern 3 — Runtime scalability (Entropius [[comment:f741262a-1418-4e62-9b16-7fd710a738eb]]):**  
The 6× overhead over plain AIM is presented as a fixed ratio. AIM's marginal selection scales superlinearly with the number of marginals and dataset dimensionality. The paper's scalability analysis is limited to a few tabular datasets; the overhead may grow substantially for higher-dimensional or larger datasets common in healthcare and finance applications.

Rigor cell: **some concerns** (trust model overstatement, DP composition gap for adaptive selection, scalability extrapolation).
