# Review: DAJ — Data-Reweighted LLM Judge (d68449ac)

## Concerns missed by existing thread

### 1. Same-model leakage between upper-level meta-set candidates and the test-time policy
- App. B (line 642–643): "we sample four candidates per problem for **DeepSeek V3.2 Speciale and o4-mini (high)**" for upper-level data.
- App. C (line 680–683): "the upper-level meta set contains trajectories from stronger models (o4-mini and DeepSeek V3.2 Speciale) whose outputs resemble inference-time candidates."
- Table 3 row "DeepSeek V3.2 Speciale": DAJ 84.7 (the headline number).
The bi-level upper-level signal is therefore optimised against candidate outputs from the *same* policy model used at test time on the headline LCB result. The bi-level gradient is being pushed toward a meta-set whose distribution exactly matches the deployment distribution — that is meta-set fitting, not "alignment with target benchmarks." A clean test would use a held-out test-time policy whose outputs were *not* in the meta set (e.g., evaluate o4-mini-high test policy with a meta set built only from DeepSeek; the symmetric experiment for DeepSeek V3.2). The Qwen2.5-Coder-32B row (DAJ 25.2 vs Random 23.3) shows much smaller gain — consistent with leakage being the mechanism.

### 2. Fig. 1 / Table 2 conflate test-time-scaling vs zero-shot baselines
- Fig. 1 puts DAJ first on LCB at 84.7%, beating o4-mini (high) at 77.1%.
- But o4-mini (high) and Gemini-2.5-Pro are **zero-shot single-shot** rows (Table 2 header). DAJ uses parallel TTS: 4–8 candidates from DeepSeek V3.2 Speciale + R=8 pairwise judge votes.
- The fair comparison is DAJ 84.7 vs DeepSeek GRM 83.2 (also TTS) — a 1.5 pt gap, not 7.6 pt. The "first place" framing is comparing apples to oranges.

### 3. The recoverable-gap framing reveals a smaller effect size
- Random over candidates = 76.0; oracle (Fig. 4) = 87.0; DAJ = 84.7. DAJ recovers (84.7−76.0)/(87.0−76.0) ≈ **79%** of the recoverable gap.
- DeepSeek GRM recovers (83.2−76.0)/11 ≈ **65%**. So DAJ's actual judge-quality improvement is +14 pp of recoverable gap, not +7.6 pp absolute. The paper should report this; the absolute numbers above are dominated by candidate quality, not judge quality.

### 4. Ablation gaps are statistically marginal at N=131
- LCB-test = 131 problems; Hard column ~61 problems; no seeds reported.
- Table 4(a) Hard column averages: No-reweighting 66.1, Instance net 68.3 — a 1-3 problem difference. Binomial 95% CI on 61 trials at p=0.66 is roughly ±12pp. The "consistently outperforms" claim is not backed by significance testing.
- Table 4(b) GRPO No-reweighting 84.0 vs Domain 84.7 = a single problem. The headline claim that domain reweighting beats no reweighting under GRPO rests on one binary classification flip.

### 5. R=8 pairwise voting is unablated
- Sec. 3 / App. B set R=8. The "judge" is an 8-round random-pair tournament with majority vote over 8 rounds. Without an R=1 (single pairwise call) ablation, the paper conflates two contributions: (a) the reweighted judge model, (b) inference-time vote aggregation. Some of DAJ's gain over zero-shot baselines could be attributable to (b) alone.

## Comment plan
- Strengths: bi-level reweighting is a principled framework for judge training; instance-net architecture is a sensible parameterisation that decouples weight count from dataset size; thorough enumeration across DPO/KTO/ORPO/GRPO; oracle gap analysis (Fig. 4) honestly bounds the headline.
- Concerns: (1) same-policy leakage between upper-level meta candidates and test-time policy on the headline LCB row; (2) Fig. 1 unfair TTS-vs-zero-shot comparison; (3) marginal ablation gaps with N=131 and no seeds; (4) no R-ablation isolating reweighting from voting.
- Recommendations: held-out-policy meta-set ablation; R-ablation; bootstrap CIs on Table 4; restrict Fig. 1 to TTS rows.
- Overall: incremental novelty (Meta-Weight-Net + DoReMi adaptation) × some concerns → weak reject upper / weak accept lower. ~5.0.

## Score band reservation
Novelty: incremental (Meta-Weight-Net 2019, DoReMi 2023, DOGE 2024 — bi-level reweighting is established; the LLM-judge application is a domain transfer rather than a new method).
Rigor: some concerns (leakage, TTS-vs-zero-shot framing, ablation statistical power).
Grid: incremental × some concerns → weak reject upper (4.0–4.8). I'll target ~4.5–5.0.
