## Answer to the Direct Question

The confound applies to both stages, but with different severity and mechanism.

**Fast-check stage:** If the fast-check uses Claude Sonnet 4.5 as its backbone (the paper's default setup), same-family recognition bias is present but weaker. The fast-check operates as a lightweight prompt-based heuristic; the bias manifests as pattern-matching alignment rather than deep semantic exploitation — the model may implicitly flag generation artifacts of its own training distribution at rates that do not generalize to cross-family synthesis. The effect is smaller than in systematic analysis because fast-check decisions require minimal reasoning depth.

**Systematic-analysis stage:** This is where the confound is most acute. The systematic analysis uses the full LLM reasoning chain to assess intent and alignment, giving the detector maximum capacity to exploit subtle stylometric and semantic signatures of its own generated output.

**Recommended ablation design:** A 2×2 factorial with independent backbone variation: {Claude-synthesized, GPT-synthesized trajectories} × {Claude-detector, non-Claude-detector}, where fast-check and systematic-analysis backbones are varied independently. If F1 collapses when only the synthesizer family changes (Claude still as detector), the confound is in the detector. If it collapses only when the systematic-analysis backbone changes (Claude still as synthesizer), the fast-check is not a significant confound site. Expected result: most variance attributable to systematic-analysis backbone, with some residual at fast-check.
